# Changelog

Все значимые изменения фиксируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).
Проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [1.0.1] — 2026-03-03

### Changed

- Таблицы отгрузок и позиций теперь отображают динамическую ширину столбцов по данным
- Разделитель `─` растягивается ровно по ширине таблицы (was: фикс. 78)

### Removed

- Мёртвый код: `_hr()`, `_BORDER`, `MoySkladClient.delete()`, `MoySkladClient.put()`, `_raise_for_status()`

## [1.0.0] — 2026-03-03

### Added

- Получение списка отгрузок за произвольный период с фильтром по статусу (`--days`, `--state-name`)
- Интерактивный выбор отгрузки по номеру из таблицы
- Сортировка позиций по ячейке склада (`slot.name`) с натуральным порядком (A-1-2 < A-1-10 < B-1-1)
- Fallback на пользовательский атрибут товара если `slot` отсутствует (`--cell-attr`)
- Применение отсортированного порядка через batch-DELETE + POST bulk (2 запроса вместо N+1)
- Верификация после сохранения: сравнение числа строк, суммарного кол-ва, суммы и порядка позиций
- Token bucket rate limiter (40 req/3s) + retry при 429/5xx/Timeout (экспоненциальный backoff, до 5 попыток)
- Флаг `--apply` для автоподтверждения без интерактивного ввода
- Флаг `--debug` для диагностического вывода сырых ответов API
- Цветной вывод таблиц (colorama)
- Авторизация через системное хранилище ключей (keyring), токен не хранится в файлах

### Technical

- `POST /entity/demand/{id}/positions/delete` — batch-удаление позиций одним запросом (endpoint подтверждён)
- `demand["id"]` используется напрямую (не парсинг `meta.href`) для устойчивости к query-параметрам
